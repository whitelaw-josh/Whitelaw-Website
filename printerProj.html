3D Printer Device Driver

  • This project was for the CMPSC 311 Systems Programming course. We developed the code to run the device driver for a 3D printer that extrudes multiple materials at certain positions. The first phase of the project was to write functions to initialize the 3D printer once it is turned on, power off the 3D printer, move the printing nozzle, retrieve the current coordinates of the printing nozzle, and to extrude the material based on data from a file listing the materials at specific coordinates to print efficiently. All of these were based on pointers that were sent to each function and decoded to reveal data such as the command to use one of these functions and the appropriate data to go with it as well.


    The second phase was to implement a file system to read data from files, store the data in allocated pointers based on offsets (for each data type), and send them to the printer drivers. As an extra, we also implemented a function to write the data that was printed after the allocated pointer was sent to the driver.


    The final phase of the project was to use socket IO and network the 3D printer. The data was sent via network and recoded in order to for the printer driver to read the data appropriately. After all the instructions were sent, then the POWEROFF command was sent to turn off the printer and the server connection was severed.
